Fix multi-company account display

Account codes show "False" in multi-company MIS reports on Odoo 18

The Bug

In Odoo 18, account.code became company-dependent. When a MIS report pulls data from multiple companies, account detail rows display "False" instead of the account code (e.g. "600000 Expenses") because the lookup reads from the wrong company context.

Mode 1: Comparison Separate columns per company

Configuration
Comparison Mode checked — each company gets its own column
Report output (after fix)
Account codes 600000 Expenses and 211000 Account Payable render correctly — not "False"

Mode 2: Totaled Consolidated single column

Configuration
Comparison Mode off, Multiple companies enabled with Branch Office + YourCompany
Report output (after fix)
Data consolidated from both companies. Account codes correct, no "False" values.
